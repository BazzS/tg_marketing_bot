Telegram Ads Agent — это программа, которая управляет рекламными кампаниями как маркетолог: следит за метриками и сама подстраивает настройки, чтобы реклама обходилась дешевле.

Что делает:

- Запускает кампании с объявлениями и бюджетами
- Каждый час собирает метрики: показы, клики, конверсии, CPC, CPL
- Сам двигает ставки — если кампания хорошо конвертит, поднимает; если деньги тратятся впустую, снижает (алгоритм Thompson Sampling)
- Перераспределяет бюджет — забирает деньги у слабых кампаний и отдаёт сильным
- Ставит на паузу кампании, у которых цена клика вышла за лимит
- Тестирует объявления A/B: запускает два варианта, ждёт статистики, оставляет лучший
- Спрашивает GPT каждые N циклов: тот смотрит на общую картину и даёт стратегические советы
- Сейчас работает на симуляторе (мок-платформа с реалистичным поведением аукциона). Для подключения реального Telegram Ads достаточно реализовать один интерфейс (AdPlatformClient).



Проект состоит из пяти слоёв.

- Платформа — абстрактный интерфейс для общения с рекламной системой. Сейчас подключён симулятор, который реалистично имитирует аукцион: ставка влияет на показы, CTR у каждого объявления свой и падает со временем, бюджет расходуется с учётом времени суток. Чтобы подключить реальный Telegram Ads API — достаточно написать один класс на том же интерфейсе.

- База данных — SQLite. Хранит кампании, объявления, метрики по каждому циклу, историю всех решений и статус A/B тестов.

- Оптимизатор — ядро самообучения. Состоит из четырёх частей: BidOptimizer двигает ставки по алгоритму Thompson Sampling, накапливая статистику конверсий; BudgetAllocator считает эффективность кампаний и перекидывает бюджет от слабых к сильным; ABTester запускает два объявления параллельно и по χ²-тесту определяет победителя; RuleEngine применяет жёсткие правила — например, ставит на паузу при превышении лимита CPC.

- LLM-стратег — раз в N циклов отправляет в GPT сводку по всем кампаниям и получает оценку тренда и рекомендации. Работает как советник: логирует выводы, но не применяет их автоматически. Без ключа OpenAI агент функционирует полностью.

- Агент и отчёты — оркестратор, который управляет циклом: собрать метрики → оптимизировать → при необходимости спросить LLM. После завершения формирует финальный отчёт с итоговыми цифрами по каждой кампании и историей решений.
